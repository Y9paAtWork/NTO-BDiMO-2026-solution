# BDiMO 2026: решение задачи категоризации товаров

## 1) Полное описание задачи

### Постановка

Нужно предсказать для каждого товара из `test.tsv` два таргета:

- `department_id` (верхний уровень категории),
- `category_id` (подкатегория).

### Формат данных

Входные признаки:

- `vendor_name`
- `vendor_code`
- `title`
- `description`
- `shop_category_name`


### Метрика

Качество решения оценивается по формуле:

$$
score = 30 \cdot F + 70 \cdot A
$$

- $F$ = macro-F1 по `department_id`,
- $A$ = accuracy по `category_id`.

### Ограничения среды проверки

- до 10 минут на сборку/прогон,
- лимиты по ресурсам: 2 CPU, 2 GB RAM,
- без GPU,
- размер репозитория ограничен.

## 2) Датасет
 
| Свойство | Значение |
|----------|----------|
| Число объектов | 8605 |
| Уникальных `category_id` | 4582 |
| Уникальных `department_id` | 16 |
| Объектов на категорию | 1-3 |
| Категории с 1 объектом | 666 (14.5%) |

При этом, `category_id -> department_id` может является однозначным соответствием, то есть category_id, будет соответствовать только 1 department_id, так что:
- можно предсказывать категорию,
- затем согласовывать/восстанавливать department по словарю категории,
- и использовать department как ограничение пространства поиска.

### Качество и шум в признаках

- Доля пустого `vendor_code`: около **40%**
- Доля пустого `shop_category_name`: около **20%**.
- `description` содержит HTML/escaped-разметку (`<br/>`, html entities), длинные шаблонные тексты и служебные блоки.
- Присутствуют разные языки/алфавиты и нестабильная нормализация написаний.

### Почему это сложно

- Классов очень много, примеров на класс очень мало.
- Много пропусков и неоднородность текстов.
- Простая многоклассовая модель в такой конфигурации обычно проседает.

## 3) Полное описание решения из этой папки

## Общая идея

Решение состоит из трех уровней:

1. Построение текстовых представлений (sparse TF-IDF + компактный dense encoder).
2. Предсказание `department_id` и `category_id` по центроидам.
3. Несколько override-правил по ключам (vendor/title/shop-category/vendor_code), которые исправляют модель там, где есть сильные табличные сигналы.

### 3.1 Подготовка текста

Функция `clean_text`:

- `html.unescape`,
- удаление HTML-тегов regex-ом,
- нормализация пробелов,
- lower-case.

Специализированные нормализаторы:

- `clean_vendor_code`: агрессивная фильтрация до букв/цифр,
- `canonicalize_title`: замена пунктуации и цифр на канонический вид.

### 3.2 Формирование признаков

#### Sparse view (основной канал)

Собирается строка поиска с повторениями полей (ручные веса через дублирование):

- `vendor_name` x1,
- `vendor_code` x1,
- `title` x5,
- `shop_category_name` x5,
- `description` x1.

Далее строятся две TF-IDF матрицы, которые позже конкатенируются в общую:

- word n-grams `(1, 2)`, `max_features=100000`,
- char_wb n-grams `(3, 5)`, `max_features=100000`.

Также используется multilingual e5-small через модуль `dense_encoder.py`, но в компактной форме:

- токенный словарь прунингуется под реально встречающиеся токены + базовый префикс,
- encoder обрезается до `DENSE_NUM_HIDDEN_LAYERS=3`,
- веса сохраняются в FP16.

Пулинг: mean pooling + L2-нормализация.

### 3.3 Обучение (`fit_bundle`)

1. Загружается train и формируются sparse/dense признаки.
2. Обучается `LinearSVC` для `department_id`:
	- `class_weight='balanced'`,
	- `C=0.4`.
3. Для `category_id` строятся центроиды в sparse-пространстве:
	- усреднение векторов объектов каждой категории через sparse assignment matrix,
	- одновременно строится маппинг category->department и индексы категорий внутри департаментов.
4. Для dense-признаков строятся отдельные центроиды:
	- по категориям,
	- по департаментам.
5. Готовится набор override-словарей по правилам (ниже).

### 3.4 Предсказание department

На инференсе:

1. Считаются скоры `LinearSVC`.
2. Прибавляется dense-вклад по центроидам департаментов с весом 0.2.
3. Для каждого объекта берутся top-k департаментов (`k=8`).
4. Объект считается неопределённым, если разница top1-top2 меньше порога (который равен 0.5).

### 3.5 Предсказание category

Скор категории:

$$
S_{cat} = S_{sparse} + w_{dense} \cdot S_{dense}
$$

где в текущей конфигурации `w_dense = 0.12`.

Далее:

- для уверенных объектов поиск ограничивается категориями top-1 департамента,
- для неопределённых объектов перебираются top-k департаментов и выбирается лучший скор.

Это сильно сокращает пространство кандидатов и стабилизирует retrieval.

### 3.6 Override правила

После базового решения применяется набор правил (первое срабатывание побеждает):

1. (`shop_category_name`, `title`)
2. (`vendor_code`,)
3. (`vendor_name`, `vendor_code`)
4. (`shop_category_name`, `vendor_code`)
5. (`vendor_name`, `title`)
6. (`shop_category_name`,)
7. (`title`,)

Для каждого правила строится мапа key->category только по уникальным соответствиям в train.

### 3.7 Согласование department и category

Если category получена через override, department принудительно берется из category_department_map.

Если category получена retrieval-веткой и department расходится, выполняется дополнительная проверка.

### 3.8 Локальная валидация

- для категорий с 3 объектами: 1 в валидацию, 2 в train,
- для категорий с 2 объектами: 1 в валидацию, 1 в train,
- категории с 1 объектом остаются только в train.

Это снижает число пустых категорий в валидации и дает более устойчивую оценку.

## Файлы проекта

- `fit/train.py` - обучение, локальная валидация, сохранение.
- `solution.py` - вся логика признаков, обучения, предсказания, скоринга.
- `dense_encoder.py` - построение и исполнение компактного dense-энкодера.
- `main.py` - инференс в контестовой среде.
- `Dockerfile` - контейнер для запуска `python3 main.py`.
- `requirements.txt` - зависимости.

---

## Как запустить

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Обучение и сохранение модели

```bash
python fit/train.py
```

Полезные параметры:

```bash
python fit/train.py --skip-validation
python fit/train.py --n-splits 5 --test-size 0.2 --random-state 42
```

### 3. Инференс на test.tsv

Положите `test.tsv` в корень папки `bdaml` и запустите:

```bash
python main.py
```

Результат: файл `prediction.csv` с колонками `department_id, category_id`.
